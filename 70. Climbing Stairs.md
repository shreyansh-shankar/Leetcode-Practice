## Pattern

👉 Dynamic Programming (1D DP)

---

## Recognition Trigger

Look for:

```text
Count number of ways
Count possible paths
Count combinations
Reach destination
```

Think:

```text
Can I build answer for n
from answers of smaller n?
```

---

# Key Insight

Suppose:

```text
n = 5
```

To reach stair 5:

### Last move was 1 step

```text
... → stair 4 → stair 5
```

Number of ways:

```text
ways(4)
```

---

### Last move was 2 steps

```text
... → stair 3 → stair 5
```

Number of ways:

```text
ways(3)
```

---

Therefore:

```text
ways(5)
=
ways(4)
+
ways(3)
```

---

# Recurrence Relation

```cpp
dp[i] = dp[i-1] + dp[i-2];
```

This is the entire problem.

---

# Why?

Every valid path to stair `i` must come from:

```text
i - 1
```

or

```text
i - 2
```

No other possibilities exist.

---

# Example

```text
n = 5
```

Base cases:

```text
dp[0] = 1
dp[1] = 2
```

Meaning:

```text
1 stair:
[1]

2 stairs:
[1+1]
[2]
```

---

Build:

```text
dp[2] = 3
dp[3] = 5
dp[4] = 8
```

Answer:

```text
8
```

---

# Your Solution

```cpp
vector<int> dp(n);

dp[0] = 1;
dp[1] = 2;

for (int i = 2; i < n; i++) {
    dp[i] = dp[i-1] + dp[i-2];
}

return dp[n-1];
```

Perfect DP solution.

---

# Important Observation

The sequence becomes:

```text
1
2
3
5
8
13
21
...
```

This is:

```text
Fibonacci
```

So Climbing Stairs is basically:

```text
Fibonacci disguised as a staircase problem
```

# Optimization

You don't actually need the entire DP array.

Because:

```cpp
dp[i]
```

only depends on:

```cpp
dp[i-1]
dp[i-2]
```

So two variables are enough.

```cpp
int a = 1;
int b = 2;

for (...) {
    int c = a + b;
    a = b;
    b = c;
}
```

Space becomes:

```text
O(1)
```

## Common Mistake

Trying to generate all paths:

```text
1+1+1+1+1
1+2+1+1
2+2+1
...
```

This becomes exponential.

DP counts paths without generating them.

## One-line Memory Hook

```text
Count ways to reach i = ways to reach (i-1) + ways to reach (i-2)
```
