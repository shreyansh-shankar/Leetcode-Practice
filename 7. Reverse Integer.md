## The New Part: Overflow

The challenge is:

```text
INT_MAX = 2147483647
INT_MIN = -2147483648
```

Suppose:

```text
rev = 214748365
```

Next step:

```cpp
rev * 10
```

already exceeds INT_MAX.

---

Therefore we must check **before multiplying**.

---

## Your Overflow Check

```cpp
if (rev > INT_MAX / 10 ||
    rev < INT_MIN / 10)
    return 0;
```

This catches most overflow cases and is accepted on LeetCode.

---

## Fully Precise Version

Many people write:

```cpp
if (rev > INT_MAX/10 ||
   (rev == INT_MAX/10 && digit > 7))
    return 0;

if (rev < INT_MIN/10 ||
   (rev == INT_MIN/10 && digit < -8))
    return 0;
```

because:

```text
2147483647
          ^
          7
```

and

```text
-2147483648
           ^
           -8
```

are the boundary digits.

---

## Accepted Code

```cpp
class Solution {
public:
    int reverse(int x) {

        int rev = 0;

        while (x != 0) {

            int digit = x % 10;
            x /= 10;

            if (rev > INT_MAX / 10 ||
                rev < INT_MIN / 10)
                return 0;

            rev = rev * 10 + digit;
        }

        return rev;
    }
};
```

## Personal Learning

This problem teaches an important interview principle:

```text
Check for overflow BEFORE performing the operation.
```

Not:

```cpp
rev = rev * 10;
if (overflow)
```

because by then overflow has already happened.
