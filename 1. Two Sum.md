## Pattern

Hash Map (Complement Lookup)

## Recognition Trigger

Look for:

- Find two numbers satisfying a condition.
- Pair sum equals target.
- Need fast lookup of a matching value.
- Pair search in an unsorted array.

Common forms:

```text
Two Sum
Pair Sum
Target Sum
Find Complement
```

When you see:

```text
a + b = target
```

think:

```text
Complement Lookup
```

---

## Key Insight

Instead of searching for both numbers:

```text
nums[i] + nums[j] = target
```

fix one number:

```text
nums[i]
```

and ask:

```text
What number do I need?
```

which is:

```cpp
needed = target - nums[i];
```

Now the problem becomes:

```text
Have I seen "needed" before?
```

which hashing solves in O(1).

---

## State Representation

```cpp
unordered_map<int, int> m;
```

Where:

```text
key   = value from array
value = index of that value
```

---

## Algorithm

For each number:

1. Compute:

```cpp
needed = target - nums[i];
```

2. Check whether:

```cpp
needed
```

already exists in the map.

3. If yes:

```text
Answer found.
```

4. Otherwise store:

```cpp
nums[i]
```

for future lookups.

---

## Accepted Code

```cpp
#include <unordered_map>

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {

        unordered_map<int, int> m;

        for (int i = 0; i < nums.size(); i++) {

            int needed = target - nums[i];

            if (m.count(needed))
                return {m[needed], i};

            m[nums[i]] = i;
        }

        return {};
    }
};
```

## Why It Works

For every number:

```text
x
```

the only value that matters is:

```text
target - x
```

Hashing allows us to instantly determine whether that complement has already appeared.

---

## Complexity

### Hash Map Solution

```text
Time: O(n)
Space: O(n)
```

---

### Brute Force

```cpp
for i
    for j > i
```

```text
Time: O(n²)
Space: O(1)
```

---

## Common Mistakes

### 1. Insert Before Checking

Wrong:

```cpp
m[nums[i]] = i;

if (m.count(needed))
```

For problems that allow duplicates, this can incorrectly use the same element twice.

Always:

```text
Check first
Then insert
```

---

### 2. Using a Set Instead of a Map

A set can tell us:

```text
Value exists
```

but not:

```text
At which index?
```

The problem requires indices.

Therefore:

```cpp
unordered_map<int,int>
```

is needed.

---

### 3. Sorting First

Sorting destroys original indices.
Extra work is then required to recover them.
