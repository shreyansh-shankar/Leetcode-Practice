## Pattern

👉 Monotonic Stack (Next Greater Element + Index Distance)

---

## Key Idea

For each day:

> “How many days until a warmer temperature appears?”

So for each index `i`, we want:

```text
next index j such that temp[j] > temp[i]
answer = j - i
```

---

# 🧠 What your stack stores

```cpp
stack<int> st;
```

It stores:

```text
indices of temperatures
```

NOT values.

Because we need:

```text
distance = j - i
```

---

# ⚙️ Core Logic

You process from **right → left**.

Why?

Because when you're at `i`, all future answers are already known.

---

## Maintain invariant:

```text
stack is strictly decreasing in temperature
```

Meaning:

```text
top of stack = next warmer candidate
```

---

# 🔥 Step-by-step logic

For each day `i`:

### 1. Remove useless future days

```cpp
while (!st.empty() && tem[st.top()] <= tem[i])
    st.pop();
```

Why?

Because:

```text
If future day is colder or equal → never useful
```

---

### 2. If stack not empty → we found answer

```cpp
result[i] = st.top() - i;
```

Because:

```text
st.top() = nearest warmer day
```

---

### 3. Push current day

```cpp
st.push(i);
```

Now it becomes a candidate for earlier days.

---

# 📊 Example

```text
temps = [73, 74, 75, 71, 69, 72, 76, 73]
```

---

### Output:

```text
[1,1,4,2,1,1,0,0]
```

---

# 🧠 Why this works

The stack always keeps:

```text
"future days that are still useful"
```

We remove anything that is:

```text
<= current temperature
```

Because they will NEVER be the answer for current or previous elements.

---

# ⚡ Complexity

### Time:

```text
O(n)
```

Each index is pushed and popped once.

### Space:

```text
O(n)
```

Stack + result array.

# 🧠 Core Insight (very important)

You are learning a general template:

```cpp
for (i from right to left):
    while stack invalid:
        pop

    answer using stack.top()

    push i
```

This is **THE monotonic stack template**.
