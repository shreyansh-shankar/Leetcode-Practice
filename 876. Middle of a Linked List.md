## Pattern

Fast and Slow Pointers (Tortoise and Hare)

## Recognition Trigger

Look for:

- Middle of a linked list
- Halfway point
- Split a linked list into two parts
- Find a position relative to the end without knowing length

Common problems:

```text
Middle of Linked List
Linked List Cycle
Cycle Start
Happy Number
Palindrome Linked List
```

When one pointer needs to reach a special position while traversing only once, think:

```text
Fast & Slow Pointers
```

---

## Key Insight

Move:

```text
slow = 1 step
fast = 2 steps
```

When fast reaches the end:

```text
slow has travelled half the distance
```

Therefore:

```text
slow points to the middle node
```

---

## State Representation

```cpp
slow
fast
```

Where:
- `slow` moves one node at a time.
- `fast` moves two nodes at a time.

---

## Algorithm

1. Start both pointers at the head.
2. Move:
    - `slow` by one step.
    - `fast` by two steps.
3. Stop when:
    - `fast == nullptr`
    - OR `fast->next == nullptr`
4. Return `slow`.

---

## Accepted Code

```cpp
class Solution {
public:
    ListNode* middleNode(ListNode* head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
        }

        return slow;
    }
};
```

## Even Length Lists

Example:

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

Result:

```text
4
```

The problem specifically asks for:

```text
the second middle node
```

Fast/Slow naturally gives this.

---

## Why It Works

Suppose:

```text
slow speed = 1
fast speed = 2
```

When fast covers:

```text
n nodes
```

slow covers:

```text
n / 2 nodes
```

Thus when fast reaches the end:

```text
slow is at the middle
```

---

## Complexity

- Time: O(n)
- Space: O(1)

---

## Common Mistakes

### 1. Counting Length First

Two-pass solution:

```text
Count nodes
↓
Traverse again to middle
```

Works, but unnecessary.

---

### 2. Incorrect Loop Condition

Wrong:

```cpp
while (fast)
```

because:

```cpp
fast->next->next
```

may dereference nullptr.

Safer:

```cpp
while (fast && fast->next)
```

---

### 3. Forgetting Even-Length Behavior

This problem wants:

```text
second middle
```

Fast/Slow automatically returns it.

## Personal Learning

This is the first problem where:

```text
Two pointers move at different speeds.
```

The important idea is:

```text
Distance relationship
```

not merely having two pointers.

Fast & Slow pointers allow us to find positions relative to the list length without explicitly computing the length.
