## Pattern

Tree DFS (Recursive Divide & Conquer)

## Recognition Trigger

Look for:
- Height of tree    
- Depth of tree    
- Longest path from root to leaf
- Information that depends on both subtrees

Common problems:

```text
Maximum Depth
Minimum Depth
Balanced Binary Tree
Diameter of Binary Tree
Maximum Path Sum
```

When the answer for a node depends on answers from its children, think:

```text
Recursive DFS
```

---

## Key Insight

Ask:

```text
What is the maximum depth of this node?
```

For any node:

```text
depth(node)
=
1 + max(
    depth(left),
    depth(right)
)
```

The problem for the whole tree becomes the same problem for smaller trees.

---

## State Representation

There is no explicit state variable.

The recursive function itself represents the state:

```cpp
maxDepth(node)
```

returns:

```text
Maximum depth of subtree rooted at node
```

---

## Recursive Formula

### Base Case

```cpp
if (root == nullptr)
    return 0;
```

An empty tree has depth 0.

---

### Recursive Case

```cpp
1 + max(
    maxDepth(root->left),
    maxDepth(root->right)
)
```

Current node contributes:

```text
1 level
```

plus the deeper subtree.

---

## Accepted Code

```cpp
class Solution {
public:
    int maxDepth(TreeNode* root) {

        if (root == nullptr)
            return 0;

        if (root->left == nullptr &&
            root->right == nullptr)
            return 1;

        return 1 + max(
            maxDepth(root->left),
            maxDepth(root->right)
        );
    }
};
```

---

## Cleaner Version

The leaf check is unnecessary.

```cpp
class Solution {
public:
    int maxDepth(TreeNode* root) {

        if (root == nullptr)
            return 0;

        return 1 + max(
            maxDepth(root->left),
            maxDepth(root->right)
        );
    }
};
```

Because:

```text
Leaf node
```

already becomes:

```text
1 + max(0,0)
=
1
```

## Why It Works

Every node asks:

```text
What is the deepest path below me?
```

Its answer depends only on:

```text
left subtree answer
right subtree answer
```

This naturally forms a recursive structure.

---

## Complexity

Let:

```text
n = number of nodes
```

### Time

```text
O(n)
```

Each node is visited once.

### Space

```text
O(h)
```

where:

```text
h = tree height
```

due to recursion stack.

Worst case:

```text
O(n)
```

for a skewed tree.

Balanced tree:

```text
O(log n)
```

## Common Mistakes

### 1. Thinking Trees Need Special Logic

Many beginners try:

```text
Many pointers
Many conditions
Complicated traversal
```

Instead ask:

```text
What should the function return for a subtree?
```

---

### 2. Forgetting Base Case

Without:

```cpp
if (root == nullptr)
```

recursion never terminates.

---

### 3. Using Global Variables

Not needed.

The answer can be built entirely from return values.

## Pattern Template

Many tree problems follow:

```cpp
ReturnType dfs(TreeNode* node) {

    if (node == nullptr)
        return baseCase;

    auto left = dfs(node->left);
    auto right = dfs(node->right);

    return combine(left, right);
}
```

104 is one of the simplest examples of this template.

## Personal Learning

The important realization is:

```text
A tree problem can often be solved by asking:

"What answer should this node return
to its parent?"
```

For this problem:

```text
Return:
Maximum depth of my subtree
```

Once that is defined, the recursive formula becomes obvious:

```text
1 + max(leftDepth, rightDepth)
```
