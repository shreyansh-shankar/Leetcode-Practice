## Pattern

👉 Prefix Sum (Running Sum)

## Recognition Trigger

Look for:

```text id="la1"
Starting value given
Changes between positions
Current value after applying gains/losses
Highest/lowest value reached
```

Think:

```text id="la2"
Running Sum / Prefix Sum
```

# Problem Understanding

You start at:

```text id="la3"
altitude = 0
```

Each element in `gain` represents:

```text id="la4"
change in altitude
```

Example:

```text id="la5"
gain = [-5,1,5,0,-7]
```

Means:

```text id="la6"
0
→ -5
→ -4
→ 1
→ 1
→ -6
```

Highest altitude:

```text id="la7"
1
```

---

# Key Insight

We do NOT need to store all altitudes.

Just maintain:

```text id="la8"
current altitude
```

and

```text id="la9"
highest altitude seen so far
```

---

# Your Solution

```cpp id="la10"
class Solution {
public:
    int largestAltitude(vector<int>& gain) {

        int currHeight = 0;
        int maxAlt = 0;

        for (auto it : gain) {
            currHeight += it;
            maxAlt = max(currHeight, maxAlt);
        }

        return maxAlt;
    }
};
```

Clean and optimal.

# Why This Is Prefix Sum

The altitude after step `i` is:

```text id="la19"
gain[0] + gain[1] + ... + gain[i]
```

That's literally:

```text id="la20"
prefix sum
```

You are just computing it on the fly.
