## Pattern

Fast & Slow Pointers (Floyd's Cycle Detection)

## Recognition Trigger

Look for:

- Detect a cycle    
- Loop detection
- Repeated traversal    
- Infinite traversal possibility
- Circular linked list

Common variants:

```text
Linked List Cycle
Linked List Cycle II
Happy Number
Find Duplicate Number
```

When a structure may loop back into itself, think:

```text
Fast & Slow Pointers
```

---

## Key Insight

Move:

```text
slow = 1 step
fast = 2 steps
```

If there is:

```text
No Cycle
```

then:

```text
fast reaches nullptr
```

and traversal ends.

---

If there is:

```text
A Cycle
```

then:

```text
fast eventually catches slow
```

inside the cycle.

Just like two runners on a circular track:

- Slow runner: 1 step
- Fast runner: 2 steps    

The faster runner must eventually lap the slower runner.

---

## State Representation

```cpp
slow
fast
```

Where:

- `slow` moves one node.    
- `fast` moves two nodes.

---

## Cleaner Implementation

```cpp
class Solution {
public:
    bool hasCycle(ListNode *head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;

            if (slow == fast)
                return true;
        }

        return false;
    }
};
```

---

## Why It Works

Suppose the cycle length is:

```text
k
```

Each iteration:

```text
fast gains 1 node on slow
```

because:

```text
2 - 1 = 1
```

Eventually:

```text
distance between them mod k
```

becomes:

```text
0
```

and they meet.

## Brute Force Alternative

Store visited nodes:

```cpp
unordered_set<ListNode*>
```

If a node is visited twice:

```text
Cycle exists.
```

Complexity:

```text
Time: O(n)
Space: O(n)
```

Fast/Slow is better because it uses:

```text
O(1) space
```

## Common Mistakes

### 1. Incorrect Loop Condition

Instead of:

```cpp
while (fast != nullptr)
```

prefer:

```cpp
while (fast && fast->next)
```

This guarantees:

```cpp
fast->next->next
```

is safe.

---

### 2. Comparing Values Instead of Nodes

Wrong:

```cpp
slow->val == fast->val
```

Different nodes can have identical values.

Need:

```cpp
slow == fast
```

---

### 3. Forgetting Empty List Cases

The standard loop condition handles:

```text
head == nullptr
head->next == nullptr
```

automatically.

## Personal Learning

The important observation is:

```text
If two pointers move at different speeds
inside a cycle,
they must eventually meet.
```

This transforms cycle detection from:

```text
O(n) extra memory
```

to:

```text
O(1) extra memory
```

using only pointer movement.
