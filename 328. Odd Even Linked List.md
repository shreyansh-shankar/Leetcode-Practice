## Pattern

👉 Linked List Pointer Manipulation

(Partition into two chains, then merge)

---

## Recognition Trigger

Look for:

```text id="oe1"
Rearrange linked list
Group nodes by some property
Do it in-place
Preserve relative order
```

Think:

```text id="oe2"
Build multiple chains using pointers
Then connect them
```

---

# Important Clarification

This problem is about:

```text id="oe3"
node positions
```

NOT values.

Example:

```text id="oe4"
1 -> 2 -> 3 -> 4 -> 5
```

Positions:

```text id="oe5"
1st  2nd  3rd  4th  5th
```

Result:

```text id="oe6"
1 -> 3 -> 5 -> 2 -> 4
```

---

# Key Insight

The list naturally forms two chains:

```text id="oe7"
Odd positions:
1 -> 3 -> 5 -> ...

Even positions:
2 -> 4 -> 6 -> ...
```

Build both chains separately and join them at the end.

---

# Your Approach

Maintain:

```cpp id="oe8"
odd
even
head2
```

Where:

```text id="oe9"
odd  = tail of odd chain
even = tail of even chain
head2 = start of even chain
```

---

# Initial Setup

Example:

```text id="oe10"
1 -> 2 -> 3 -> 4 -> 5
```

```cpp id="oe11"
odd = head       (1)
even = head->next (2)
head2 = even
```

---

# Main Loop

```cpp id="oe12"
odd->next = even->next;
odd = odd->next;
```

Connect odd nodes:

```text id="oe13"
1 -> 3
```

Move odd tail.

---

Then:

```cpp id="oe14"
even->next = even->next->next;
even = even->next;
```

Connect even nodes:

```text id="oe15"
2 -> 4
```

Move even tail.

---

# Visual Walkthrough

Initial:

```text id="oe16"
1 -> 2 -> 3 -> 4 -> 5
O    E
```

---

After first iteration:

```text id="oe17"
Odd chain:
1 -> 3

Even chain:
2 -> 4

Remaining:
5
```

---

After second iteration:

```text id="oe18"
Odd chain:
1 -> 3 -> 5

Even chain:
2 -> 4
```

Loop ends.

---

# Final Step

Connect chains:

```cpp id="oe19"
odd->next = head2;
```

Result:

```text id="oe20"
1 -> 3 -> 5 -> 2 -> 4
```

---

# Why Store `head2`?

Because:

```text id="oe21"
even keeps moving
```

If we don't save the start of the even list:

```text id="oe22"
2 -> 4 -> 6
```

we lose access to it.

So:

```cpp id="oe23"
head2 = even;
```

stores the head of the even chain.

---

# Key Invariant

During execution:

```text id="oe24"
odd points to tail of odd chain

even points to tail of even chain
```

Both chains remain valid linked lists.

---

# Complexity

### Time

```text id="oe25"
O(n)
```

Each node visited once.

---

### Space

```text id="oe26"
O(1)
```

Only pointers used.

## Core Template

```cpp id="oe30"
odd = head
even = head->next
evenHead = even

while (...) {
    odd->next = even->next
    odd = odd->next

    even->next = odd->next
    even = even->next
}

odd->next = evenHead
```


## Common Mistake

Confusing:

```text id="oe32"
odd/even values
```

with

```text id="oe33"
odd/even positions
```

The problem uses:

```text id="oe34"
positions
```

only.
