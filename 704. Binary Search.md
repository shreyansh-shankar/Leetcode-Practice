## Pattern

👉 Binary Search

---

## Recognition Trigger

Look for:

```text
sorted array
search for target
O(log n)
```

Immediately think:

```text
Binary Search
```

---

# Key Insight

Since the array is sorted:

```text
If nums[mid] > target
→ target can only be on the left

If nums[mid] < target
→ target can only be on the right
```

So every comparison eliminates:

```text
half of the search space
```

---

# Your Solution

```cpp
while (high >= low) {

    mid = low + (high - low) / 2;

    if (nums[mid] > target) {
        high = mid - 1;
    }
    else if (nums[mid] < target) {
        low = mid + 1;
    }
    else {
        return mid;
    }
}

return -1;
```

This is the standard implementation.

---

# Example

```text
nums = [-1,0,3,5,9,12]
target = 9
```

---

### Iteration 1

```text
low = 0
high = 5

mid = 2
nums[mid] = 3
```

Since:

```text
3 < 9
```

Move right:

```text
low = 3
```

---

### Iteration 2

```text
low = 3
high = 5

mid = 4
nums[mid] = 9
```

Found.

```text
return 4
```

---

# Why

```cpp
mid = low + (high - low) / 2;
```

instead of:

```cpp
mid = (low + high) / 2;
```

?

Because:

```text
low + high
```

can overflow for very large arrays.

This version is overflow-safe.

---

Time:

```text
O(log n)
```

Because each step removes half the remaining elements.

## Core Template

```cpp
int l = 0;
int r = n - 1;

while (l <= r) {

    int mid = l + (r - l) / 2;

    if (nums[mid] == target)
        return mid;

    else if (nums[mid] < target)
        l = mid + 1;

    else
        r = mid - 1;
}

return -1;
```
