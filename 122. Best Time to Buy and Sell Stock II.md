## Pattern

Greedy

## Recognition Trigger

- Unlimited transactions are allowed.
- No transaction fee.
- No cooldown period.
- Goal is to maximize total profit.

When every profitable transaction can be taken independently, think:

```text
Take every positive gain.
```

---

## Key Insight

Every upward price movement contributes to the optimal answer.

Example:

```text
1 2 3 4 5
```

Can be viewed as:

```text
Buy at 1
Sell at 5
Profit = 4
```

or

```text
Buy at 1 Sell at 2 = 1
Buy at 2 Sell at 3 = 1
Buy at 3 Sell at 4 = 1
Buy at 4 Sell at 5 = 1

Total = 4
```

Same answer.

Therefore, we never need to wait for the "perfect" sell point.

We can safely collect every positive increase.

---

## State Representation

```cpp
curr
profit
```

Where:
- `curr` = effective buying price / previous price in the current trend.
- `profit` = total profit accumulated so far.

---

## Algorithm

1. Start with the first price as the current buy price.    
2. Traverse the array.
3. If price decreases:
    - Update the buy price.
4. If price increases:
    - Add the profit immediately.
    - Treat today's price as the new buy price.

---

## Accepted Code

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int profit = 0;
        int curr = prices[0];

        for (int i = 1; i < prices.size(); i++) {
            if (curr > prices[i]) {
                curr = prices[i];
            }
            else if (curr < prices[i]) {
                profit += prices[i] - curr;
                curr = prices[i];
            }
        }

        return profit;
    }
};
```

---

## Alternative Compact Form

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int profit = 0;

        for (int i = 1; i < prices.size(); i++) {
            if (prices[i] > prices[i - 1]) {
                profit += prices[i] - prices[i - 1];
            }
        }

        return profit;
    }
};
```

---

## Why It Works

Every increasing sequence:

```text
1 -> 3 -> 5 -> 8
```

contributes:

```text
(3-1) + (5-3) + (8-5)
```

which equals:

```text
8 - 1
```

Therefore taking profits early never hurts the final answer.

## Complexity

- Time: O(n)
- Space: O(1)

## Common Mistake

Trying to find the single best buy/sell pair like Problem 121.

That solves:

```text
At most one transaction
```

but this problem allows:

```text
Unlimited transactions
```

which changes the optimal strategy completely.

## Personal Learning

The difference between 121 and 122 is only one constraint:

```text
ONE transaction
↓
UNLIMITED transactions
```

Yet the optimal solution changes completely.

Always pay close attention to constraints before choosing a pattern.
