## Pattern

Sliding Window + Last Seen Index

## Recognition Trigger

Look for:
- Longest/shortest subarray or substring
- Contiguous segment
- Constraint must be maintained
- Characters entering and leaving a window

Common examples:

```text
Longest Substring Without Repeating Characters
Longest Repeating Character Replacement
Permutation in String
Minimum Window Substring
Fruit Into Baskets
```

When you see:

```text
Longest valid contiguous segment
```

think:

```text
Sliding Window
```

## Key Insight

Brute force would check every substring:

```text
O(n²)
```

Instead maintain a window:

```text
[left ... right]
```

such that:

```text
All characters are unique.
```

When a duplicate appears:

```text
Move left boundary forward.
```

## Why This Condition Is Necessary

You wrote:

```cpp
if (m[s[i]] + 1 > maxLen.first)
    maxLen.first = m[s[i]] + 1;
```

This is extremely important.

Example:

```text
abba
```

At final:

```text
a
```

last seen:

```text
0
```

Current window:

```text
ba
```

left boundary:

```text
2
```

If you blindly do:

```cpp
left = m['a'] + 1;
```

you would move:

```text
2 → 1
```

which is backwards.

Not allowed.

Therefore:

```cpp
left = max(left, m[c] + 1);
```

is required.

Your condition is doing exactly that.

## Cleaner Equivalent

Your code is basically equivalent to:

```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {

        unordered_map<char,int> lastSeen;

        int left = 0;
        int ans = 0;

        for (int right = 0; right < s.size(); right++) {

            if (lastSeen.count(s[right])) {
                left = max(left,
                           lastSeen[s[right]] + 1);
            }

            lastSeen[s[right]] = right;

            ans = max(ans,
                      right - left + 1);
        }

        return ans;
    }
};
```

## Common Mistakes

### 1. Moving Left Backwards

Wrong:

```cpp
left = lastSeen[c] + 1;
```

Need:

```cpp
left = max(left,
           lastSeen[c] + 1);
```

---

### 2. Erasing Characters One By One

Many people do:

```cpp
while (duplicate)
    shrink window
```

Works.

But slower and more complicated.

The last-seen-index trick jumps directly.

---

### 3. Forgetting To Update Last Seen Index

Need:

```cpp
lastSeen[c] = i;
```

every time.

Not only on first appearance.

The most important insight is:

```text
When a duplicate appears,
don't move left one step at a time.

Jump directly to:
(last occurrence + 1)
```

The hashmap is not storing frequency.

It is storing:

```text
Character → Most Recent Position
```

which allows direct window adjustment.
