## Key Insight

We already know how to add numbers manually:

```text
  342
+ 465
-----
  807
```

Process digits:

```text
2 + 5 = 7
4 + 6 = 10
carry = 1
3 + 4 + 1 = 8
```

The problem simply asks us to perform the same process while traversing linked lists.

## Core Formula

For each digit:

```cpp
sum = digit1 + digit2 + carry;
```

Current digit:

```cpp
sum % 10
```

New carry:

```cpp
sum / 10
```

## Cleaner Version

Most solutions combine all loops into one:

```cpp
class Solution {
public:
    ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {

        ListNode dummy(0);
        ListNode* tail = &dummy;

        int carry = 0;

        while (l1 || l2 || carry) {

            int sum = carry;

            if (l1) {
                sum += l1->val;
                l1 = l1->next;
            }

            if (l2) {
                sum += l2->val;
                l2 = l2->next;
            }

            carry = sum / 10;

            tail->next = new ListNode(sum % 10);
            tail = tail->next;
        }

        return dummy.next;
    }
};
```

---

## Why This Is Nice

Instead of:

```text
Both exist
Only l1 exists
Only l2 exists
Carry remains
```

all cases are handled by:

```cpp
while (l1 || l2 || carry)
```

One loop.

One invariant.


## Common Mistakes

### 1. Forgetting Final Carry

Example:

```text
9 -> 9
1
```

Produces:

```text
0 -> 0 -> 1
```

Without handling final carry:

```text
0 -> 0
```

Wrong.

---

### 2. Stopping When One List Ends

Need to continue processing:

```text
Remaining digits
+
Carry
```

---

### 3. Losing the Head Pointer

Same lesson as:

**21. Merge Two Sorted Lists**

Need:

```cpp
head
```

or

```cpp
dummy node
```

to return the start.
