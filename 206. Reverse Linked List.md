## Pattern

Pointer Manipulation / In-Place Reversal

## Recognition Trigger

Look for:

- Reverse a linked list
- Reverse a chain
- Reverse nodes in-place
- Change direction of pointers

Common variations:

```text
Reverse Linked List
Reverse Linked List II
Reverse Nodes in K Group
Palindrome Linked List
Reorder List
```

When a linked list needs to be reversed, think:

```text
prev
curr
next
```

---

## Key Insight

When reversing a node:

```text
A -> B -> C
```

After changing:

```cpp
B->next = A;
```

the original link to:

```text
C
```

would be lost.

Therefore:

```text
Save next first.
Then reverse.
Then move forward.
```

---

## State Representation

```cpp
prev
curr
next
```

Where:

- `prev` = already reversed portion
- `curr` = current node being processed
- `next` = saved reference to the remaining list

---

## Algorithm

For each node:

### Step 1

Save the next node.

```cpp
next = curr->next;
```

---

### Step 2

Reverse the pointer.

```cpp
curr->next = prev;
```

---

### Step 3

Advance pointers.

```cpp
prev = curr;
curr = next;
```

Repeat until:

```cpp
curr == nullptr
```

---

## Accepted Code

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* curr = head;

        while (curr != nullptr) {
            ListNode* next = curr->next;

            curr->next = prev;

            prev = curr;
            curr = next;
        }

        return prev;
    }
};
```

## Why It Works

At every iteration:

### Reversed Portion

```text
prev
```

contains a valid reversed list.

### Unprocessed Portion

```text
curr
```

contains the remaining original list.

The invariant is maintained until the entire list is reversed.

---

## Complexity

- Time: O(n)
- Space: O(1)

---

## Common Mistakes

### 1. Forgetting to Save Next

Wrong:

```cpp
curr->next = prev;
curr = curr->next;
```

After reversing:

```text
curr->next
```

now points backward.

The rest of the list is lost.

---

### 2. Returning head

After reversal:

```cpp
head
```

still points to the old first node.

The new head is:

```cpp
prev
```

---

### 3. Moving Pointers in Wrong Order

Always:

```cpp
save next
reverse
move pointers
```

Changing the order can lose access to nodes.

## Recursive Version

```cpp
ListNode* reverseList(ListNode* head) {
    if (!head || !head->next)
        return head;

    ListNode* newHead = reverseList(head->next);

    head->next->next = head;
    head->next = nullptr;

    return newHead;
}
```

Time:

```text
O(n)
```

Space:

```text
O(n)
```

due to recursion stack.

## Personal Learning

The important lesson is not:

```text
How to reverse a linked list.
```

The important lesson is:

```text
When modifying pointers,
save access to the remaining structure first.
```

The critical state is:

```cpp
prev
curr
next
```

Store the exact pointers needed before changing links.
