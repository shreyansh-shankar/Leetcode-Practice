## Pattern

👉 Two Pointers (Stable Partition)

---

## Recognition Trigger

Look for:

```text id="mz1"
move all X to one side
keep relative order unchanged
in-place modification
```

Think:

```text id="mz2"
Two pointers
One reads
One writes
```

---

# Key Insight

We want:

```text id="mz3"
All non-zero elements first
All zeros at the end
```

while preserving order.

Example:

```text id="mz4"
[0,1,0,3,12]
```

Desired:

```text id="mz5"
[1,3,12,0,0]
```

---

# Your Approach

You maintain:

```cpp id="mz6"
zero
```

which represents:

```text id="mz7"
the next position where a non-zero element should go
```

---

# How It Works

Initially:

```cpp id="mz8"
zero = 0
```

---

## Example

```text id="mz9"
[0,1,0,3,12]
```

---

### i = 0

```text id="mz10"
nums[i] = 0
```

Ignore.

```text id="mz11"
zero = 0
```

---

### i = 1

```text id="mz12"
nums[i] = 1
```

Swap:

```cpp id="mz13"
swap(nums[0], nums[1]);
```

Array:

```text id="mz14"
[1,0,0,3,12]
```

Move write pointer:

```cpp id="mz15"
zero++;
```

```text id="mz16"
zero = 1
```

---

### i = 2

```text id="mz17"
nums[i] = 0
```

Ignore.

---

### i = 3

```text id="mz18"
nums[i] = 3
```

Swap:

```cpp id="mz19"
swap(nums[1], nums[3]);
```

Array:

```text id="mz20"
[1,3,0,0,12]
```

---

### i = 4

```cpp id="mz21"
swap(nums[2], nums[4]);
```

Array:

```text id="mz22"
[1,3,12,0,0]
```

Done.

---

# Why It Works

At every step:

```text id="mz23"
[0 ... zero-1]
```

contains:

```text id="mz24"
all non-zero elements in original order
```

And:

```text id="mz25"
[i+1 ... end]
```

has not been processed yet.

---

# Important Insight

The variable:

```cpp id="mz26"
zero
```

is slightly misleadingly named.

It's actually:

```text id="mz27"
next position for a non-zero element
```

Think of it as:

```cpp id="mz28"
writeIndex
```

and the algorithm becomes easier to understand.

---

# Complexity

### Time

```text id="mz29"
O(n)
```

Single pass.

---

### Space

```text id="mz30"
O(1)
```

In-place.

# Personal Learning Note

## Pattern

Two Pointers (Read/Write)

---

## Recognition Trigger

```text id="mz31"
Move elements matching a condition
Keep relative order
Modify array in-place
```

---

## Key Insight

Use:

```cpp id="mz32"
i        -> scans array
writeIdx -> next valid position
```

Whenever a valid element is found:

```cpp id="mz33"
swap(nums[writeIdx], nums[i]);
writeIdx++;
```

---

## Invariant

Before every iteration:

```text id="mz34"
[0 ... writeIdx-1]
contains all processed non-zero elements
in correct order
```

---

## Complexity

```text id="mz35"
Time: O(n)
Space: O(1)
```

---

## Common Mistake

Using:

```text id="mz36"
erase()
insert()
```

for every zero.

That turns the solution into:

```text id="mz37"
O(n²)
```

---

## One-line Memory Hook

```text id="mz38"
Scan with one pointer, compact non-zero elements with another.
```
