## Pattern

Heap (Priority Queue) + Lazy Removal

## Recognition Trigger

Look for:

- Maximum/Minimum in a moving window
- Dynamic range queries
- Need current largest element while elements enter and leave

Common examples:

```text
Sliding Window Maximum
Sliding Window Median
Top K Elements
K Closest Points
```

When you need:

```text
Current maximum among active elements
```

think:

```text
Max Heap
```

---

## Key Insight

The window keeps moving:

```text
[1 3 -1]
  [3 -1 -3]
     [-1 -3 5]
```

Elements:

- Enter from the right
- Leave from the left

A heap quickly gives:

```text
Largest element
```

but cannot efficiently remove arbitrary old elements.

Solution:

```text
Lazy Removal
```

Keep old elements in the heap and remove them only when they reach the top.

---

## State Representation

```cpp
priority_queue<pair<int,int>>
```

Store:

```text
(value, index)
```

The index tells us whether the element is still inside the current window.

---

## Algorithm

For every window:

### Add New Element

```cpp
pq.push({nums[i], i});
```

---

### Remove Expired Maximums

```cpp
while (pq.top().second < windowStart)
    pq.pop();
```

If the top element is outside the current window:

```text
Discard it.
```

---

### Current Maximum

```cpp
pq.top().first
```

---

## Accepted Code

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {

        vector<int> res;
        priority_queue<pair<int,int>> pq;

        for (int i = 0; i < k - 1; i++) {
            pq.push({nums[i], i});
        }

        int curr_window = 0;

        for (int i = k - 1; i < nums.size(); i++) {

            pq.push({nums[i], i});

            while (pq.top().second < curr_window)
                pq.pop();

            res.push_back(pq.top().first);

            curr_window++;
        }

        return res;
    }
};
```

---

## Why Store Indices?

Suppose:

```text
nums = [1,3,-1,-3]
```

Window:

```text
[1,3]
```

Heap top:

```text
3
```

Move window:

```text
[3,-1]
```

Still fine.

Move again:

```text
[-1,-3]
```

Heap still contains:

```text
3
```

Need the index to determine:

```text
3 is no longer inside the window.
```

---

## Complexity

### Heap Solution

Each push:

```text
O(log n)
```

Each pop:

```text
O(log n)
```

Overall:

```text
Time: O(n log n)
Space: O(n)
```

---

## Common Mistakes

### 1. Storing Only Values

Wrong:

```cpp
priority_queue<int>
```

Cannot determine whether the maximum has left the window.

Need:

```cpp
(value, index)
```

---

### 2. Removing Old Elements Immediately

Heap cannot efficiently remove arbitrary elements.

Use:

```text
Lazy Removal
```

instead.

---

### 3. Forgetting Multiple Expired Elements

Need:

```cpp
while (...)
```

not:

```cpp
if (...)
```

Several outdated elements may accumulate.

---

## Optimal Solution

Pattern:

```text
Monotonic Deque
```

Complexity:

```text
Time: O(n)
Space: O(k)
```

The deque maintains elements in decreasing order.

Front always contains:

```text
Current window maximum
```

## Personal Learning

The crucial idea is:

```text
Maximum element may leave the window.
```

Therefore simply remembering:

```text
currentMax
```

is insufficient.

We must also know:

```text
whether that maximum is still valid.
```

This is why indices are stored along with values.
