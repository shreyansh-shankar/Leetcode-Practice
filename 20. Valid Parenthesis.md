## Pattern

Stack

## Recognition Trigger

Look for:

- Matching pairs
- Nested structures
- Opening and closing symbols
- "Most recent unmatched element"

Common examples:

```text
()
[]
{}
HTML tags
XML tags
Nested expressions
Compiler parsing
```

When something must be matched in reverse order, think:

```text
Stack (LIFO)
```

---

## Key Insight

The most recently opened bracket must be the first one to close.

Example:

```text
({[]})
```

Order of opening:

```text
(
{
[
```

Order of closing:

```text
]
}
)
```

Notice:

```text
Last Opened
First Closed
```

This is exactly how a stack works.

---

## State Representation

```cpp
stack<char> st;
```

Stores all currently unmatched opening brackets.

---

## Algorithm

For each character:

### Opening Bracket

```text
(
[
{
```

Push onto stack.

---

### Closing Bracket

```text
)
]
}
```

1. Stack must not be empty.
2. Top of stack must contain the matching opening bracket.
3. Pop if matched.
4. Otherwise return false.

---

## Accepted Code

```cpp
#include <stack>
class Solution {
public:
    bool isValid(string s) {
        stack<char> st;

        for (char c: s) {
            if (c == '{' || c == '(' || c == '[') {
                st.push(c);
            } else if (c == ']' || c == '}' || c == ')') {
                if (st.empty()) return false;
                else if (c == ')') {
                    if (st.top() == '(') {
                        st.pop();
                    } else {
                        return false;
                    }
                } else if (c == ']') {
                    if (st.top() == '[') {
                        st.pop();
                    } else {
                        return false;
                    }
                } else if (c == '}') {
                    if (st.top() == '{') {
                        st.pop();
                    } else {
                        return false;
                    }
                }
            }
        }

        return st.empty();
    }
};
```

---

## Why It Works

Consider:

```text
([)]
```

Stack operations:

```text
Push (
Push [
See )

Top = [
Expected = (

Mismatch → Invalid
```

The stack immediately detects incorrect nesting.

---

## Complexity

- Time: O(n)
- Space: O(n)

Worst case:

```text
(((((((((
```

All brackets remain in the stack.

---

## Common Mistakes

### 1. Forgetting Empty Stack Check

```cpp
st.top()
```

before:

```cpp
st.empty()
```

can crash.

---

### 2. Only Checking Counts

Wrong approach:

```text
Count '('
Count ')'
```

Example:

```text
)(
```

Counts match but ordering is invalid.

---

### 3. Forgetting Final Stack Check

Example:

```text
(((
```

No mismatch occurs during traversal.

But stack is not empty.

Must return:

```text
false
```

---

## Alternative Cleaner Solution

Store expected closing brackets:

```cpp
if (c == '(') st.push(')');
if (c == '[') st.push(']');
if (c == '{') st.push('}');
```

Then:

```cpp
if (st.empty() || st.top() != c)
    return false;

st.pop();
```

This removes most of the repeated code.
