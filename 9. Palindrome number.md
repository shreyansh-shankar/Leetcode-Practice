Number Reversal / Digit Manipulation

## Recognition Trigger

Look for:
- Reverse a number
- Process digits individually
- Extract digits using `% 10`
- Build a new number digit-by-digit

Common problems:

```text
Palindrome Number
Reverse Integer
Plus One
Add Digits
Happy Number
```

When you see:

```text
Need to inspect decimal digits
```

think:

```text
Digit Manipulation
```

---

## Key Insight

A number is a palindrome if:

```text
Forward == Backward
```

Example:

```text
121
```

Reverse:

```text
121
```

Same.

Therefore:

```text
Reverse the number
Compare with original
```

---

## State Representation

```cpp
temp
```

Remaining digits to process.

---

```cpp
rev
```

Reversed number built so far.

---

## Algorithm

Repeatedly:

### Extract Last Digit

```cpp
temp % 10
```

---

### Append To Reversed Number

```cpp
rev = rev * 10 + digit;
```

---

### Remove Last Digit

```cpp
temp /= 10;
```

---

Continue until:

```cpp
temp == 0
```

---

## Accepted Code

```cpp
class Solution {
public:
    bool isPalindrome(int x) {

        int temp = x;
        long rev = 0;

        while (temp >= 1) {
            rev = rev * 10 + (temp % 10);
            temp /= 10;
        }

        return rev == x;
    }
};
```

## Common Mistakes

### 1. Ignoring Negative Numbers

Example:

```text
-121
```

Reverse:

```text
121-
```

Not valid.

Should return:

```text
false
```

Your solution already does because:

```cpp
temp >= 1
```

never runs.

---

### 2. Integer Overflow

Reversing:

```text
2147483647
```

can overflow an `int`.

That's why:

```cpp
long rev
```

is safer.

---

### 3. Converting To String Immediately

Works:

```cpp
to_string(x)
```

but misses the digit manipulation practice.

---

## Better Observation

A palindrome cannot end with:

```text
0
```

unless the number is:

```text
0
```

because:

```text
10
```

would become:

```text
01
```

which is not a valid representation.

This is used in the optimized solution.

## Personal Learning

The important takeaway is:

```text
Numbers can be treated as sequences of digits.
```

The two fundamental operations are:

```cpp
digit = n % 10;
n /= 10;
```

These appear repeatedly in math and number-manipulation problems.
