## Recognition Trigger

Look for:

```text id="rs1"
running sum
cumulative sum
prefix sum
sum up to index i
range sum queries
```

When you see:

```text id="rs2"
sum of elements from start to current position
```

think:

```text id="rs3"
Prefix Sum
```

---

# Key Insight

Instead of repeatedly summing:

```text id="rs4"
nums[0] + nums[1] + ... + nums[i]
```

store the cumulative sum as you go.

---

## Example

Input:

```text id="rs5"
[1,2,3,4]
```

Process:

```text id="rs6"
sum = 1  -> [1]
sum = 3  -> [1,3]
sum = 6  -> [1,3,6]
sum = 10 -> [1,3,6,10]
```

Output:

```text id="rs7"
[1,3,6,10]
```

---

# Your Solution

```cpp
class Solution {
public:
    vector<int> runningSum(vector<int>& nums) {

        vector<int> runSum;
        int sum = 0;

        for (auto it : nums) {
            sum += it;
            runSum.push_back(sum);
        }

        return runSum;
    }
};
```

Clean and correct.

# In-Place Optimization

Since modifying the input is allowed:

```cpp
class Solution {
public:
    vector<int> runningSum(vector<int>& nums) {

        for (int i = 1; i < nums.size(); i++) {
            nums[i] += nums[i - 1];
        }

        return nums;
    }
};
```

Space:

```text id="rs10"
O(1)
```

(extra space)

# Why Prefix Sum Matters

This problem itself is easy.

The important part is what it leads to.

Prefix sums are used in:

```text id="rs11"
Range Sum Query
Subarray Sum Equals K
Continuous Subarray Sum
Product Except Self (related idea)
2D Prefix Sum problems
```

---

# Core Formula

If:

```text id="rs12"
prefix[i] = sum of elements from 0 to i
```

then:

```text id="rs13"
sum(l...r)
=
prefix[r] - prefix[l-1]
```

This turns:

```text id="rs14"
O(n) range queries
```

into:

```text id="rs15"
O(1)
```
