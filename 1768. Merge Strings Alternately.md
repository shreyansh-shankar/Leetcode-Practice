# Cleaner Way (Standard Pattern)

Instead of a counter, most people just write both conditions:

```cpp id="m6"
class Solution {
public:
    string mergeAlternately(string word1, string word2) {

        int i = 0, j = 0;
        string res = "";

        while (i < word1.size() && j < word2.size()) {
            res += word1[i++];
            res += word2[j++];
        }

        while (i < word1.size())
            res += word1[i++];

        while (j < word2.size())
            res += word2[j++];

        return res;
    }
};
```

## Recognition Trigger

```text id="m11"

```

---

## Key Insight

Use two pointers:

```cpp id="m12"
i for word1
j for word2
```

Always:

```text id="m13"
take one from each while possible
then append leftovers
```

---

## Core Template

```cpp id="m14"
while (i < n && j < m) {
    take from A
    take from B
}

append remaining A
append remaining B
```
