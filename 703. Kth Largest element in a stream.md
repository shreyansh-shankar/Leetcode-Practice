## Pattern

👉 Heap (Fixed-Size Min Heap)

---

## Recognition Trigger

Look for:

```text id="kls1"
Kth largest
Kth smallest
Continuous stream of values
Maintain top K elements
```

Think:

```text id="kls2"
Fixed-size heap
```

---

# Key Insight

We do NOT need:

```text id="kls3"
all elements sorted
```

We only need:

```text id="kls4"
the K largest elements seen so far
```

---

## Example

```cpp id="kls5"
k = 3

Stream:
4, 5, 8, 2
```

The 3 largest are:

```text id="kls6"
4, 5, 8
```

Notice:

```text id="kls7"
2
```

is useless because it can never become part of the top 3.

---

# Why Min Heap?

Store exactly:

```text id="kls8"
K largest elements
```

inside the heap.

The smallest among them is:

```text id="kls9"
the Kth largest overall
```

So:

```cpp id="kls10"
pq.top()
```

is always the answer.

---

# How Your Solution Works

You use:

```cpp id="kls11"
priority_queue<
    int,
    vector<int>,
    greater<int>
> pq;
```

which creates a:

```text id="kls12"
Min Heap
```

---

## Add Element

```cpp id="kls13"
pq.push(val);
```

---

If heap becomes too large:

```cpp id="kls14"
if (pq.size() > k)
    pq.pop();
```

remove:

```text id="kls15"
the smallest element
```

---

After this:

```text id="kls16"
heap always contains
exactly the k largest elements seen so far
```

# Why Constructor Uses `add()`

You wrote:

```cpp id="kls31"
for (auto it : nums) {
    add(it);
}
```

Very clean.

This guarantees:

```text id="kls32"
all heap maintenance logic
exists in one place
```

instead of duplicating code.

---

# Complexity

Let:

```text id="kls33"
n = stream size
```

---

### add()

Push:

```text id="kls34"
O(log k)
```

Pop:

```text id="kls35"
O(log k)
```

Total:

```text id="kls36"
O(log k)
```

---

### Space

```text id="kls37"
O(k)
```

Heap never stores more than:

```text id="kls38"
k elements
```

# Personal Learning Note

## Pattern

Fixed-Size Heap

---

## Recognition Trigger

```text id="kls39"
Kth largest
Kth smallest
Top K elements
Streaming data
```

---

## Key Insight

Maintain:

```text id="kls40"
only K useful elements
```

instead of storing everything.

---

## Rule

For:

```text id="kls41"
Kth Largest
```

use:

```text id="kls42"
Min Heap of size K
```

For:

```text id="kls43"
Kth Smallest
```

use:

```text id="kls44"
Max Heap of size K
```

---

## Invariant

At all times:

```text id="kls45"
Heap contains exactly
the K largest elements seen so far.
```

---

## Answer

```cpp id="kls46"
pq.top()
```

because:

```text id="kls47"
smallest among top K
=
Kth largest overall
```

---

## Complexity

```text id="kls48"
add(): O(log k)

Space: O(k)
```

---

## Common Mistake

Using:

```text id="kls49"
Max Heap of all elements
```

which gives:

```text id="kls50"
O(n) space
```

and doesn't exploit the fact that only K elements matter.

---

## One-line Memory Hook

```text id="kls51"
Keep only the K largest numbers in a min heap; the heap top is always the Kth largest.
```
