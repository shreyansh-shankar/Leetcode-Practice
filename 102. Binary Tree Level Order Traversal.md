## State Representation

```cpp
queue<TreeNode*> q;
```

The queue stores:

```text
Nodes waiting to be processed
```

## Why `size = q.size()` Matters

This is the most important part of the solution.

Suppose queue contains:

```text
[9,20]
```

These are all nodes of the current level.

We do:

```cpp
int size = q.size();
```

Now:

```text
size = 2
```

Process exactly:

```text
2 nodes
```

Even though while processing them we may push:

```text
15
7
```

into the queue.

Those belong to:

```text
Next level
```

and should not be processed yet.

## Complexity

Let:

```text
n = number of nodes
```

### Time

```text
O(n)
```

Every node enters and leaves the queue once.

### Space

```text
O(w)
```

where:

```text
w = maximum width of tree
```

Worst case:

```text
O(n)
```

## About Your Code

This part:

```cpp
if (!q.front()) {
    q.pop();
    continue;
}
```

is unnecessary.

Because you only ever push:

```cpp
if (q.front()->left)
    q.push(q.front()->left);

if (q.front()->right)
    q.push(q.front()->right);
```

So:

```text
nullptr is never added to queue
```

You can safely remove that check.

---

Cleaner version:

```cpp
class Solution {
public:
    vector<vector<int>> levelOrder(TreeNode* root) {
        vector<vector<int>> result;

        if (!root) return result;

        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {

            int size = q.size();
            vector<int> level;

            for (int i = 0; i < size; i++) {

                TreeNode* node = q.front();
                q.pop();

                level.push_back(node->val);

                if (node->left)
                    q.push(node->left);

                if (node->right)
                    q.push(node->right);
            }

            result.push_back(level);
        }

        return result;
    }
};
```

## Personal Learning

### Pattern

```text
Level-by-Level Processing
→ BFS
→ Queue
```

### Core Insight

```text
Queue contains the current frontier.
```

At the start of each iteration:

```cpp
int size = q.size();
```

captures exactly:

```text
Number of nodes in current level.
```

This allows us to separate:

```text
Current level
from
Next level
```

even though both temporarily exist in the same queue.

## One-line Memory Hook

```text
Tree levels → BFS → Queue → process q.size() nodes per level
```

Also notice your pattern progression:

```text
104 Max Depth
→ DFS (Recursion)

102 Level Order Traversal

```

These are the **two fundamental tree traversal patterns**. Almost every tree problem is built on either DFS or BFS.
