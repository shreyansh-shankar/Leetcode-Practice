## Pattern

👉 Greedy (Farthest Reach)

---

## Recognition Trigger

Look for:

```text
Can I reach the end?
Maximum jump length
Reachability problem
```

Think:

```text
What's the farthest position I can reach so far?
```

---

# Key Insight

You don't actually care:

```text
Which path did I take?
```

You only care:

```text
What is the farthest index reachable right now?
```

---

# Your Approach

You maintain:

```cpp
curr
jump
```

Where:

```text
curr = current index being explored

jump = farthest index reachable so far
```

---

# Example

```cpp
nums = [2,3,1,1,4]
```

Initially:

```text
curr = 0
jump = 2
```

Meaning:

```text
I can currently reach indices [0..2]
```

---

### Explore index 1

```cpp
jump = max(2, 1 + 3)
     = 4
```

Now:

```text
I can reach up to index 4
```

Since:

```text
4 >= n-1
```

Answer:

```text
true
```

---

# What the Loop Means

```cpp
while (curr < jump)
```

This means:

```text
Keep exploring positions that are reachable.
```

The moment:

```text
curr == jump
```

and you haven't extended reach further,

you are stuck.

Example:

```cpp
[3,2,1,0,4]
```

---

Process:

```text
curr=0 jump=3

curr=1 jump=3

curr=2 jump=3

curr=3 jump=3
```

Cannot go beyond:

```text
index 3
```

Loop ends.

Return:

```text
false
```

---

# Why This Works

At every step:

```text
jump
```

stores:

```text
the farthest reachable position using any index seen so far
```

Not just the current index.

That's the greedy insight.

---

# Complexity

### Time

```text
O(n)
```

Each index is processed once.

---

### Space

```text
O(1)
```

# Personal Learning Note

## Pattern

Greedy (Farthest Reach)

---

## Recognition Trigger

```text
Can reach end?
Jump lengths
Reachability
```

---

## Key Insight

Track:

```cpp
farthestReach
```

For every reachable index:

```cpp
farthestReach =
    max(farthestReach,
        i + nums[i]);
```

---

## Invariant

Before processing index `i`:

```text
All indices <= farthestReach
are reachable.
```

---

## Core Template

```cpp
reach = 0;

for (int i = 0; i <= reach; i++) {
    reach = max(reach, i + nums[i]);

    if (reach >= n-1)
        return true;
}

return false;
```

Your solution is the same idea expressed with:

```cpp
curr
jump
```

instead of:

```cpp
i
reach
```

---

## Complexity

```text
Time: O(n)
Space: O(1)
```

---

## Common Mistake

Thinking:

```text
Which jump should I take?
```

The problem never asks for a path.

Only:

```text
Is the end reachable?
```

So track reachability, not actual jumps.

---

## One-line Memory Hook

```text
Keep extending the farthest reachable index; if exploration gets stuck before the last index, answer is false.
```
