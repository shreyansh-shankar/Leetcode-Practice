## Pattern

👉 Fast & Slow Pointer (Gap Technique)

---

## Recognition Trigger

Look for:

```text id="rn1"
Nth from end
Kth from end
Remove node near end
Single pass linked list
```

Think:

```text id="rn2"
Maintain a fixed gap between two pointers
```

---

# Key Insight

Instead of:

```text id="rn3"
finding list length
then finding target node
```

Create a gap of:

```text id="rn4"
n nodes
```

between:

```text id="rn5"
slow
fast
```

Then move both together.

# Your Approach

You create:

```cpp id="rn6"
slow = dummy node
fast = dummy node
```

Dummy:

```text id="rn7"
0 -> head
```

This is a common linked-list trick.

---

# Why Dummy Node?

Consider:

```text id="rn8"
[1]
n = 1
```

Need to remove:

```text id="rn9"
head
```

Without a dummy node:

```text id="rn10"
special case needed
```

With a dummy:

```text id="rn11"
0 -> 1
```

everything becomes uniform.

---

# Step 1 — Create Gap

```cpp id="rn12"
for (int i = 0; i < n; i++)
    fast = fast->next;
```

Example:

```text id="rn13"
1 -> 2 -> 3 -> 4 -> 5
n = 2
```

After loop:

```text id="rn14"
slow -> dummy
fast -> 2
```

Gap:

```text id="rn15"
2 nodes
```

---

# Step 2 — Move Together

```cpp id="rn16"
while (fast->next)
```

Move:

```cpp id="rn17"
slow = slow->next;
fast = fast->next;
```

until:

```text id="rn18"
fast reaches last node
```

---

At that moment:

```text id="rn19"
slow is immediately before
the node to remove
```

---

Example:

```text id="rn20"
1 -> 2 -> 3 -> 4 -> 5
           ^
          slow
```

Target:

```text id="rn21"
4
```

---

# Step 3 — Delete Node

```cpp id="rn22"
slow->next = slow->next->next;
```

This skips:

```text id="rn23"
target node
```

---

# Example

```text id="rn24"
1 -> 2 -> 3 -> 4 -> 5
n = 2
```

Remove:

```text id="rn25"
4
```

Result:

```text id="rn26"
1 -> 2 -> 3 -> 5
```

# One Small Issue in Your Code

You wrote:

```cpp id="rn27"
if (fast->next == nullptr)
    return slow->next->next;
```

This handles:

```text id="rn28"
remove head
```

separately.

Because:

```text id="rn29"
slow = dummy
```

at that moment.

---

A cleaner dummy-node version is:

```cpp id="rn30"
for (int i = 0; i <= n; i++)
    fast = fast->next;
```

Create a gap of:

```text id="rn31"
n + 1
```

Then no special case is needed.

But your solution is still correct.

## Key Insight

Keep:

```text id="rn35"
fast
```

ahead of:

```text id="rn36"
slow
```

by exactly:

```text id="rn37"
n nodes
```

When:

```text id="rn38"
fast reaches end
```

then:

```text id="rn39"
slow reaches node before target
```

---

## Core Template

```cpp id="rn40"
move fast n steps

while (fast not at end) {
    slow = slow->next;
    fast = fast->next;
}
```

---

## Remove Node

```cpp id="rn41"
slow->next = slow->next->next;
```

---

## Complexity

```text id="rn42"
Time: O(n)
Space: O(1)
```

---

## Common Mistake

Trying to:

```text id="rn43"
find length first
```

Then:

```text id="rn44"
traverse again
```

That works, but misses the intended fast/slow pointer pattern.

---

## One-line Memory Hook

```text id="rn45"
Keep fast n nodes ahead; when fast reaches the end, slow is right before the node to remove.
```
