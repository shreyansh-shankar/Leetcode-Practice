## Pattern

Two Pointers (Merge Process)

## Recognition Trigger

Look for:

- Two sorted arrays/lists
- Merge two sorted structures
- Combine while maintaining sorted order

Common problems:

```text
Merge Two Sorted Lists
Merge Sorted Array
Merge Intervals
Merge K Sorted Lists
External Merge Sort
```

When you see:

```text
Two sorted inputs
```

think:

```text
Two Pointers
```

---

## Key Insight

At any moment:

```text
The smallest remaining element
must be at the front of one of the lists.
```

Therefore:

1. Compare current nodes.
2. Take the smaller one.
3. Advance that pointer.

Repeat until one list is exhausted.

---

## State Representation

```cpp
list1
list2
tmp
head
```

Where:

- `list1` = current node in first list
- `list2` = current node in second list
- `tmp` = tail of merged list
- `head` = start of merged list

---

## Algorithm

While both lists exist:

```text
Compare front elements
Take smaller node
Advance corresponding pointer
Advance merged-list pointer
```

After one list finishes:

```text
Attach the remaining nodes
```

because they are already sorted.

---

## Accepted Code

```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {

        if (list1 == nullptr && list2 == nullptr) return nullptr;
        else if (list1 == nullptr) return list2;
        else if (list2 == nullptr) return list1;

        ListNode* head;

        if (list2->val >= list1->val) {
            head = list1;
            list1 = list1->next;
        } else {
            head = list2;
            list2 = list2->next;
        }

        ListNode* tmp = head;

        while (list1 != nullptr && list2 != nullptr) {

            if (list1->val > list2->val) {
                tmp->next = list2;
                list2 = list2->next;
            } else {
                tmp->next = list1;
                list1 = list1->next;
            }

            tmp = tmp->next;
        }

        while (list1 != nullptr) {
            tmp->next = list1;
            list1 = list1->next;
            tmp = tmp->next;
        }

        while (list2 != nullptr) {
            tmp->next = list2;
            list2 = list2->next;
            tmp = tmp->next;
        }

        return head;
    }
};
```

## Cleaner Version (Dummy Node)

A common interview version:

```cpp
ListNode dummy;
ListNode* tail = &dummy;

while (list1 && list2) {

    if (list1->val <= list2->val) {
        tail->next = list1;
        list1 = list1->next;
    } else {
        tail->next = list2;
        list2 = list2->next;
    }

    tail = tail->next;
}

tail->next = list1 ? list1 : list2;

return dummy.next;
```

Dummy nodes eliminate:

```text
Special handling of the head node.
```

---

## Why It Works

Suppose:

```text
list1: 1 -> 3 -> 7
list2: 2 -> 4 -> 5
```

First comparison:

```text
1 vs 2
```

Take:

```text
1
```

Next:

```text
3 vs 2
```

Take:

```text
2
```

Continue until both lists are consumed.

Since both lists are already sorted:

```text
The smaller front element is always globally correct.
```

This is a greedy choice.

---

## Complexity

Let:

```text
n = length of list1
m = length of list2
```

Then:

```text
Time: O(n + m)
Space: O(1)
```

(No new nodes created.)

---

## Common Mistakes

### 1. Forgetting Remaining Nodes

After:

```cpp
while (list1 && list2)
```

one list may still contain nodes.

Must append them.

---

### 2. Losing the Head Pointer

If only:

```cpp
tmp
```

is stored,

the beginning of the merged list can be lost.

Need:

```cpp
head
```

(or dummy node).

---

### 3. Creating New Nodes Unnecessarily

Not wrong, but wastes memory.

We can simply reconnect existing nodes.
