## Pattern

👉 Prefix Product + Suffix Product

---

## Recognition Trigger

Look for:

```text id="p1"
For each index:
use all elements except current one
```

or

```text id="p2"
Need left contribution + right contribution
```

Think:

```text id="p3"
Prefix + Suffix
```

---

# Key Insight

For index `i`:

```text id="p4"
answer[i]
=
(product of everything left of i)
×
(product of everything right of i)
```

---

## Example

```text id="p5"
nums = [1,2,3,4]
```

For:

```text id="p6"
i = 2 (value = 3)
```

Answer should be:

```text id="p7"
1 × 2 × 4 = 8
```

Think of it as:

```text id="p8"
left product  = 1 × 2 = 2
right product = 4

answer = 2 × 4 = 8
```

---

# Prefix Array

Store:

```text id="p9"
product from beginning up to i
```

Example:

```text id="p10"
nums   = [1,2,3,4]
prefix = [1,2,6,24]
```

Meaning:

```text id="p11"
prefix[0] = 1
prefix[1] = 1×2
prefix[2] = 1×2×3
prefix[3] = 1×2×3×4
```

---

# Suffix Array

Store:

```text id="p12"
product from i to end
```

Conceptually:

```text id="p13"
suffix = [24,24,12,4]
```

Meaning:

```text id="p14"
suffix[0] = 1×2×3×4
suffix[1] = 2×3×4
suffix[2] = 3×4
suffix[3] = 4
```

Your implementation stores suffix reversed, which works but is harder to reason about.

---

# Core Formula

For a middle element:

```cpp id="p15"
answer[i]
=
prefix[i-1]
*
suffix[i+1]
```

Because:

```text id="p16"
prefix[i-1]
= everything left

suffix[i+1]
= everything right
```

---

# Edge Cases

### First Element

No left side exists.

```cpp id="p17"
answer[0] = suffix[1];
```

---

### Last Element

No right side exists.

```cpp id="p18"
answer[n-1] = prefix[n-2];
```

---

# Your Approach

Build:

```cpp id="p19"
prefix
suffix
```

Then combine them.

Time:

```text id="p20"
O(n)
```

Space:

```text id="p21"
O(n)
```

---

# Optimal Solution

LeetCode expects:

```text id="p22"
O(n) time
O(1) extra space
```

Idea:

Store prefix directly inside answer array.

Then traverse from right while maintaining:

```cpp id="p23"
suffixProduct
```

No separate suffix array needed.

---

# Why This Problem Is Important

This is NOT really about multiplication.

This is about learning a reusable pattern:

```text id="p24"
For each index:
combine information from left side
and information from right side
```

You'll see this idea again in:

```text id="p25"
Trapping Rain Water
Maximum Product Subarray
Prefix/Suffix Sum problems
Prefix XOR problems
```

## Common Mistake

Trying:

```text id="p31"
totalProduct / nums[i]
```

Fails when:

```text id="p32"
nums contains 0
```

That's why the problem explicitly forbids division.
