## Pattern

Running Minimum / Prefix Minimum

## Recognition Trigger

- Need to maximize a difference.
- One element must come before another (`buy before sell`).
- For every current element, only the best previous element matters.

Typical form:

max(arr[j] - arr[i]) where i < j

---

## Key Insight

For every selling day, the only buying day worth considering is the cheapest price seen so far.

Instead of checking all previous days:

profit_today = current_price - minimum_price_seen_so_far

---

## State Representation

- `minPriceSoFar` = smallest price encountered up to the current day.
- `maxProfit` = best profit found so far.

---

## Algorithm

1. Initialize `minPriceSoFar` with the first price.
2. Traverse the array once.
3. Update `minPriceSoFar` if a lower price is found.
4. Otherwise calculate:
    - `currentProfit = price - minPriceSoFar`
5. Update `maxProfit` if `currentProfit` is larger.

---

## Accepted Code

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int minPriceSoFar = prices[0];
        int maxProfit = 0;

        for (int i = 1; i < prices.size(); i++) {

            if (prices[i] < minPriceSoFar) {
                minPriceSoFar = prices[i];
            }

            else if (prices[i] - minPriceSoFar > maxProfit) {
                maxProfit = prices[i] - minPriceSoFar;
            }
        }

        return maxProfit;
    }
};
```

---

## Complexity

- Time: O(n)
- Space: O(1)

---

## Common Mistake

Brute force:

```cpp
for (i)
    for (j > i)
```

Time complexity becomes O(n²).

---

## Mental Model

While scanning the array:

- Keep remembering the cheapest stock seen so far.
- At each day ask:

"If I sell today, what is the best profit I can make?"

## Personal Learning

This solution works because I stored exactly the information I needed:

- Minimum price seen so far

instead of storing:

- All previous prices
- Buy day / sell day combinations

A lot of optimal solutions come from identifying the minimum state needed to make the next decision.
